# MDX Vacuum for Astro

This project provides an example of how magic the use of [MDX](https://mdxjs.com) in [Astro](https://astro.build) can be, to create beautiful and function-rich web sites, even for non-tech editors.

_MDX Vacuum_ provides tools to setup your publishing pipeline based on **clean** MDX, using [Astro's MDX support](https://docs.astro.build/en/guides/markdown-content/).

On NPM you'll find all [`mdxvac` plugins](https://www.npmjs.com/org/mdxvac).

## Clean MDX = Magic MDX

With clean I mean markdown, with only a few sprinkles of JSX and without overloading the frontmatter.

<Claim>Less `X` ... More Magic*</Claim>

## Check out the existing plugins

<Wrap>
  <Card title="Frontmatter" href="/plugins/remark-astro-frontmatter">

  <Card title="Element Mapping" href="/plugins/vite-astro-mdxcomponents">

    components on a per-directory basis.
  </Card>
  <Card title="Raw MDX Content" href="/plugins/remark-astro-rawmdx">
    Plugin `remark-astro-rawmdx` adds the raw MDX content to the frontmatter, to analyze in the
    layout, e.g. to compute reading time.
  </Card>
  <Card title="Auto Import" href="/plugins/remark-astro-autoimports">
    Plugin `remark-astro-autoimports` lets users define a set of components that can be used in
    MDX-files without explicitly being imported.
  </Card>
  <Card title="Sections for Headings" href="/plugins/remark-sectionize-headings">
    Plugin `remark-sectionize-headings` wraps headings and the following paragraphs into sections,
    e.g. to style them with CSS' `nth-child(even)`.
  </Card>
</Wrap>

## Discuss upcoming plugins

- **remark-astro-title-abstract** - extracts the title and abstract of a markdown file and adds it to the frontmatter `title` and `abstract` properties.
- **remark-astro-import-src** - imports files relative to referencing file from string `src` attributes.
- **remark-astro-style-directive** - lets users apply CSS classes to the HTML elements resulting from the markdown.
- **remark-astro-mdast** - adds the markdown AST (`mdast`) to the frontmatter, to analyze in the layout, e.g. to generate a TOC.
- **remark-astro-hast** - adds the HTML AST (`hast`) to the frontmatter, to analyze in the layout, e.g. to ???.

## Deprecated

- **remark-astro-auto-layout** - despite being the most successful plugin thus far, you should use the `remark-astro-frontmatter` plugin instead to define your common layout.
